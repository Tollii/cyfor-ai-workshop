# AGENTS.md

## Repository overview

This repository is a small npm workspaces monorepo for a workshop application.

- `api/`: Hono REST API, Prisma ORM, SQLite database, OpenAPI export
- `web/`: React + Vite frontend using Orval-generated React Query hooks
- `slides/`: standalone Reveal.js presentation assets
- `workshop-tasks/`: markdown task briefs for participants

The root workspace manages `api` and `web`. Most application changes should happen in one of those two folders.

## Prerequisites

- Node.js 20+
- npm 10+

Install dependencies from the repository root:

```bash
npm install
```

## Common commands

Run these from the repository root unless there is a specific reason to scope to a workspace.

```bash
npm run dev
npm run build
npm run typecheck
npm run generate
```

What they do:

- `npm run dev`: starts the API and web app together. The API defaults to port `3000`. The web app waits for `http://127.0.0.1:${API_PORT:-3000}/health` before starting.
- `npm run build`: builds all workspaces that define a build script.
- `npm run typecheck`: type-checks all workspaces that define a typecheck script.
- `npm run generate`: runs generation in all workspaces that define it.

Useful scoped commands:

```bash
npm run dev --workspace api
npm run dev --workspace web
npm run build --workspace api
npm run build --workspace web
```

Slides are not part of the npm workspaces. To preview them locally:

```bash
cd slides
npx serve . -l 3030
```

## Architecture notes

### API

- Entry point: `api/src/index.ts`
- Main route and OpenAPI definitions: `api/src/app.ts`
- Database schema: `api/prisma/schema.prisma`
- SQLite database file: `api/data/workshop.db`
- OpenAPI export script: `api/scripts/export-openapi.ts`

The API serves:

- `GET /`
- `GET /health`
- `GET /items`
- `POST /items`
- `DELETE /items/{id}`
- `GET /openapi.json`

Prisma client generation and schema sync are part of the API startup flow. Deleting `api/data/workshop.db` and restarting the API recreates the local database.

### Web

- App entry point: `web/src/App.tsx`
- API client barrel: `web/src/api/index.ts`
- Custom Orval HTTP client: `web/src/api/client.ts`
- Generated hooks: `web/src/api/generated/hooks.ts`
- Orval config: `web/orval.config.ts`

The frontend expects the API locally and proxies `/api` requests during development.

## Generated files and edit boundaries

Do not hand-edit generated artifacts unless the task is specifically about generation output.

- `api/openapi.json` is generated from the API route definitions in `api/src/app.ts`.
- `web/src/api/generated/hooks.ts` is generated by Orval from `api/openapi.json`.

For API contract changes:

1. Update `api/src/app.ts` and, if needed, `api/prisma/schema.prisma`.
2. Run `npm run generate` from the repository root.
3. Re-run the relevant build or typecheck command.

When changing frontend API usage, prefer editing `web/src/App.tsx`, `web/src/api/client.ts`, or other non-generated source files.

## Validation guidance

There is no dedicated test suite in the current repository structure. Use the narrowest available validation for the area you change:

- `npm run typecheck`
- `npm run build`
- `npm run generate`
- manual verification with `npm run dev`

For targeted checks during API work:

```bash
npm run build --workspace api
```

For targeted checks during web work:

```bash
npm run typecheck --workspace web
npm run build --workspace web
```

## Environment and runtime notes

- Override the API port with `API_PORT` when needed.
- Set `CORS_ORIGIN` to a comma-separated allowlist if the frontend is hosted separately from the API.
- Set `VITE_API_BASE_URL` for a built frontend that should call a non-default API origin.

## Practical guidance for agents

- Prefer minimal, local edits that preserve the workshop starter structure.
- Check whether a file is generated before editing it.
- If a change affects the API contract, regenerate artifacts before validating.
- Use root scripts first so workspace interactions stay consistent.
- Keep slides and workshop task documents separate from application logic unless the task explicitly targets them.
